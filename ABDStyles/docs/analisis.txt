🔍 Los 2 Algoritmos Clave de ABDAgRAG
El código implementa dos funciones de utilidad que debemos portar tal cual a la suite por su genialidad matemática:

1. Cálculo de Contraste Inteligente (Fórmula YIQ de Accesibilidad)
Evita que si un Tenant elige color primario amarillo, el texto del botón se pinte de blanco (haciéndolo ilegible).

typescript
function getContrastColor(hexcolor: string) {
  if (!hexcolor || !hexcolor.startsWith('#')) return '#ffffff';
  try {
    const r = parseInt(hexcolor.substring(1, 3), 16);
    const g = parseInt(hexcolor.substring(3, 5), 16);
    const b = parseInt(hexcolor.substring(5, 7), 16);
    const yiq = ((r * 299) + (g * 587) + (b * 114)) / 1000;
    return (yiq >= 128) ? '#000000' : '#ffffff'; // Devuelve texto negro o blanco según el fondo
  } catch (e) {
    return '#ffffff';
  }
}
2. Ajuste Dinámico para Modo Oscuro (Hex Bitwise Arithmetic)
Evita que si el color primario es azul marino oscuro, en el modo oscuro el sitio sea una mancha negra ilegible. Ajusta la luminancia dinámicamente mediante operaciones a nivel de bits:

typescript
function adjustColor(hex: string, percent: number) {
  if (!hex || !hex.startsWith('#')) return hex;
  try {
    const num = parseInt(hex.replace("#", ""), 16),
      amt = Math.round(2.55 * percent),
      R = (num >> 16) + amt,
      G = (num >> 8 & 0x00FF) + amt,
      B = (num & 0x0000FF) + amt;
    return "#" + (0x1000000 + (R < 255 ? R < 1 ? 0 : R : 255) * 0x10000 + (G < 255 ? G < 1 ? 0 : G : 255) * 0x100 + (B < 255 ? B < 1 ? 0 : B : 255)).toString(16).slice(1);
  } catch (e) {
    return hex;
  }
}

En ABDAgRAG, el BrandingProvider realizaba un fetch('/api/tenant/branding') en el lado del cliente. Esto causaba un pequeño parpadeo (la aplicación cargaba con el tema azul por defecto por un milisegundo y de pronto cambiaba a los colores del Tenant tras completarse la petición de red).

Paso 1: Crea el Helper de Branding en el Satélite
Crea un archivo utilitario en src/lib/tenant-branding.ts de ABDQuiz con los algoritmos matemáticos y un generador de plantilla CSS:

typescript
// src/lib/tenant-branding.ts
export function generateTenantCss(colors: { primary?: string; secondary?: string }) {
  const primary = colors.primary || '#0f172a';
  const primaryFg = getContrastColor(primary);
  const primaryDark = adjustColor(primary, 20);
  const primaryDarkFg = getContrastColor(primaryDark);
  return `
    :root {
      --color-primary: ${primary} !important;
      --color-primary-foreground: ${primaryFg} !important;
      --color-ring: ${primary} !important;
      ${colors.secondary ? `--color-secondary: ${colors.secondary} !important;` : ''}
    }
    .dark {
      --color-primary: ${primaryDark} !important;
      --color-primary-foreground: ${primaryDarkFg} !important;
      --color-ring: ${primaryDark} !important;
    }
  `;
}

🎨 El Motor de Conversión Hexadecimal a HSL Dinámico
Dado que los tenants introducirán colores en formato Hexadecimal (ej. #ef4444) pero Tailwind v4 los lee en formato de componentes HSL plano (H S% L%), necesitamos una función de conversión de alta precisión.

He diseñado la siguiente utilidad server-side para realizar esta transformación en tiempo de ejecución:

typescript
// src/lib/color-utils.ts
/**
 * Convierte un color Hexadecimal plano a la cadena de componentes HSL compatible con Tailwind CSS v4.
 * Ejemplo: "#3b82f6" -> "217 91.2% 59.8%"
 */
export function hexToHslComponents(hex: string): string {
  // Saneamiento de entrada
  hex = hex.replace(/^#/, '');
  if (hex.length === 3) {
    hex = hex.split('').map(char => char + char).join('');
  }
  const r = parseInt(hex.substring(0, 2), 16) / 255;
  const g = parseInt(hex.substring(2, 4), 16) / 255;
  const b = parseInt(hex.substring(4, 6), 16) / 255;
  const max = Math.max(r, g, b);
  const min = Math.min(r, g, b);
  let h = 0;
  let s = 0;
  const l = (max + min) / 2;
  if (max !== min) {
    const d = max - min;
    s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
    switch (max) {
      case r: h = (g - b) / d + (g < b ? 6 : 0); break;
      case g: h = (b - r) / d + 2; break;
      case b: h = (r - g) / d + 4; break;
    }
    h /= 6;
  }
  const hDeg = Math.round(h * 360);
  const sPct = Math.round(s * 100);
  const lPct = Math.round(l * 100);
  return `${hDeg} ${sPct}% ${lPct}%`;
}

📝 Checklist de Operaciones y Seguridad: Lo que debes tener en cuenta en Producción
Para que el despliegue de esta marca blanca dinámica en Vercel sea un éxito rotundo, existen 4 aspectos clave de seguridad e infraestructura que debes anticipar:

1. 🛡️ Sanitización de Entradas (Prevención de Inyección de CSS)
El Riesgo: Si permites que un administrador de un Tenant guarde cualquier cadena de texto en el campo del color (por ejemplo, en lugar de un color introduce body { display: none } o una URL maliciosa que haga phishing), al inyectarlo directamente en el <style> del layout comprometerías la seguridad de tu suite.
La Mitigación: Valida de forma estricta los campos de color en el backend usando Zod antes de guardarlos en MongoDB. Solo admite formatos hexadecimales puros:
typescript
const themeSchema = z.object({
  primary: z.string().regex(/^#[0-9a-fA-F]{6}$/, "Debe ser un Hexadecimal válido (ej. #3b82f6)"),
  secondary: z.string().regex(/^#[0-9a-fA-F]{6}$/).optional(),
  rounded: z.boolean()
});
2. 📦 Almacenamiento de Logotipos en Vercel (Storage Ephemeral)
El Riesgo: El sistema de archivos de Vercel (Serverless Functions) es efímero y de solo lectura. No puedes guardar los logotipos subidos por los Tenants en una carpeta local como public/logos/. Al cabo de unos minutos, el servidor se destruirá y los archivos desaparecerán.
La Mitigación: Debes subir las imágenes a un servicio de almacenamiento en la nube. La solución más rápida, integrada y gratuita en sus primeros niveles para Vercel es Vercel Blob Storage (@vercel/blob). Permite subir archivos directamente desde Next.js con 3 líneas de código y te devuelve una URL CDN pública ultra-rápida segura (https://xxxx.public.blob.vercel-storage.com/logo.png).
3. 🌐 Detección de Tenant por Subdominio en Vercel
El Riesgo: En local usas localhost:3300, pero en producción querrás que bomberos.abdquiz.com cargue la marca de bomberos y policia.abdquiz.com cargue la de policía.
La Mitigación:
En Vercel: Configura un dominio comodín o Wildcard Domain (*.abdquiz.com) apuntando a tu despliegue.
En el Código: En tu proxy.ts (middleware), puedes extraer el subdominio leyendo la cabecera host del request:
typescript
const host = request.headers.get('host') || '';
const subdomain = host.split('.')[0]; // "bomberos"
En Local: Puedes probar los subdominios fácilmente usando el dominio de bucle de retorno lvh.me (que apunta siempre a 127.0.0.1). Si entras en bomberos.lvh.me:3300, tu código detectará el subdominio bomberos igual que en producción.
4. 🦺 Favicon Dinámico (Cuidado con la Caché del Navegador)
El Riesgo: Los navegadores guardan en caché los favicons de forma extremadamente agresiva. Si un usuario cambia de Tenant en la misma pestaña, el favicon a veces no se actualiza.
La Mitigación: Al inyectar el favicon dinámicamente en el cliente (como hace la función en BrandingProvider.tsx de ABDAgRAG), añádele un parámetro de consulta aleatorio (cache buster) basado en la versión del tema para forzar la recarga:
typescript
link.href = `${favicon.url}?v=${themeVersion}`;
