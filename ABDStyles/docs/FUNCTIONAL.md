# Functional Specifications Guide - `@ajabadia/styles`

This document details the functional specifications, product roadmap, and operational guidelines for the **Dynamic Multi-Tenant Customizer** platform across the ABD SaaS suite.

---

## 🎨 Functional Vision: White-Label SaaS Architecture

As the ABD suite scales, each tenant (academy, business, or organization) must feel that the platform is their own exclusive product. This is achieved by allowing administrators to seamlessly customize their brand parameters via a centralized **Branding Customizer Console** in `ABDAuth` / `ABDGovernance`.

```
[ Tenant Admin Console ] 
   │
   ├── Primary Color Picker (Hex) ──────> [ Live Simulator Box ]
   ├── Secondary Color Picker (Hex) ────> [ Real-time visual updates ]
   └── Drag & Drop Logo Image ──────────> [ Uploads to Vercel Blob ]
```

---

## 🖥️ Branding Customizer UI Console

The central administration panel will expose a highly interactive, premium UI component: **`TenantBrandingForm.tsx`** designed following strict visual aesthetic guidelines (sleek dark mode, glassmorphism, precise borders).

### Key Console Capabilities:
1.  **Color Selectors**: Styled color-pickers (`<input type="color">`) allowing exact Hex values for the brand's primary and secondary elements.
2.  **Logo File Area (Drag & Drop)**: A lightweight, secure file selector supporting standard image assets (PNG, JPEG, WebP, SVG).
3.  **Visual Live Preview Simulator**:
    *   An embedded iframe or mini mock-component simulating an active exam session from `ABDQuiz`.
    *   When the admin changes a color or drops a logo, the preview component reflects the changes **instantly** (reacting in real-time using local React state before saving).
4.  **Radius Controller**: A toggle or selector to switch between a modern curved aesthetic (`rounded-[0.75rem]`) and a sharp, ciber-industrial aesthetic (`rounded-none`).

## ☁️ Media Asset Pipelines (Vercel Blob Storage)

Serverless functions on Vercel are **ephemeral** and read-only. Consequently, brand logos uploaded by tenants cannot be saved directly into a local directory like `public/logos/`.

### The Upload Workflow:
1.  The Tenant Admin uploads their corporate logo through `TenantBrandingForm.tsx`.
2.  The backend server-action uploads the image buffer securely to **Vercel Blob Storage** (`@vercel/blob`) under a dedicated bucket:
    ```typescript
    import { put } from '@vercel/blob';
    const { url } = await put(`logos/${tenantId}-logo.png`, file, { access: 'public' });
    ```
3.  The API returns a highly-performant CDN URL (e.g. `https://xxxx.public.blob.vercel-storage.com/logos/tenant-logo.png`).
4.  This CDN URL is persisted in the MongoDB Tenant document inside `branding.logoUrl`.

## 🌐 Dynamic Tenant Routing & Local Testing

In production, the system determines the active tenant by reading the request subdomains (e.g., `bomberos.abdquiz.com` or `policia.abdquiz.com`).

### 1. Subdomain Resolution in Middleware
A shared routing middleware extracts the active tenant key from the HTTP `Host` header:
```typescript
const host = request.headers.get('host') || '';
const subdomain = host.split('.')[0]; // "bomberos"
```
The application then fetches the branding parameters associated with this subdomain from the database.

### 2. Local Host Loopback Testing (`lvh.me`)
For seamless local development, testing subdomains can be achieved by using the loopback domain **`lvh.me`** which always maps to `127.0.0.1`.
*   Configure local listening on port `3300` (`ABDQuiz`) or `3400` (`ABDAuth`).
*   Access `bomberos.lvh.me:3300` or `policia.lvh.me:3300` to simulate full multi-tenant subdomain resolution in a local environment.

## 🦺 Favicon Dynamics & Cache-Busting

Web browsers cache favicons aggressively, which can lead to stale logos when users navigate between different tenant portals.

To prevent this issue, the HTML `<link rel="icon">` generated by the engine includes a cache-busting query parameter linked to the last modification date or version of the tenant's brand configuration:

```html
<link rel="icon" href="${branding.faviconUrl}?v=${themeVersion}" />
```
This forces the browser to bypass its internal cache and display the correct corporate icon immediately.
